ros主从机配置

2021-05-07ROS

master就是主机，主机启动roscore 之后从机就连的主机的roscore

一般配好了之后两边都能互相收到节点和话题，如果出现节点图有红的，那就是没配好的。

.bashrc

车：vscode打开.bashrc 添加
export ROS_HOSTNAME=sz-Default-string       # 小车电脑名，在小车终端底下hostname就行
export ROS_MASTER_URI=http://192.168.5.101:11311    # http://小车电脑IP:11311，在小车终端底下ifconfig看一下

pc：vscode打开.bashrc （zsh的开zshrc） 添加
export ROS_HOSTNAME=bender巴拉巴拉巴拉    # PC电脑名，在pc终端底下hostname放进去
export ROS_MASTER_URI=http://192.168.5.101:11311      # http://小车电脑IP:11311在小车终端底下ifconfig看一下
export ROS_IP="192.168.5.xxxxx"                 #在自己电脑的终端底下ifconfig看一下

hosts

车：车的终端底下，sudo nano /etc/hosts   添加 
192.168.5.101    sz-Default-string                 #　小车IP（在小车终端底下ifconfig看一下）　小车名（在小车终端底下hostname看一下）
192.168.5.xxxxx    bender巴拉巴拉巴拉        #  PC_IP（在PC终端底下ifconfig看一下）  PC名（在PC终端底下ifconfig看一下）

pc:打开etc目录下的hosts文件 添加
192.168.5.101    sz-Default-string                 #　小车IP（在小车终端底下ifconfig看一下）　小车名（在小车终端底下hostname看一下）
192.168.5.xxxxx    bender巴拉巴拉巴拉        #  PC_IP（在PC终端底下ifconfig看一下）  PC名（在PC终端底下ifconfig看一下）

rosapp的图像显示问题

之前有个课设用rosapp，手机上可以控制cmdvel，但是收不到图像，配置也是跟之前配主从机一样，如果没记错的话好像是因为我们之前用的是hostname是那种自己电脑字符那样的benderbalabla，然后hosts里设置解析了一下这个bender是192.168.balabalabala….后来我们改成hostname直接写这个域名而不是字符，就省去解析这个过程，好使了。。虽然不太知道原理。。有人说可能是动静态ip的问题？
